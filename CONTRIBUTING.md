## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/mikasamoto/TeleVault.git
   cd TeleVault
   ```
3. **Install dependencies:**
   ```bash
   npm install
   ```
4. **Run the app locally:**
   ```bash
   npm start
   ```
5. Create a new branch for your change:
   ```bash
   git checkout -b feat/your-feature-name
   ```

## How to Contribute

You can contribute in many ways — not just code:

- Fix bugs or typos
- Improve documentation or the README
- Suggest or implement new features
- Test the app and report issues
- Help others in Issues/Discussions
- Translate the app or docs

---

## Reporting Bugs

Before opening an issue, please:

- Search existing issues to avoid duplicates
- Make sure you're on the latest version

When reporting, include:

- Your OS and Node.js version
- Steps to reproduce the bug
- What you expected vs what actually happened
- Screenshots or logs if applicable

Open a bug report → [New Issue](../../issues/new)

## Suggesting Features

Feature requests are welcome. Open an issue with:

- A clear description of the feature
- Why it would be useful
- Any examples or mockups if you have them

Please don't open a pull request for a large feature without discussing it in an issue first.

---

## Pull Request Guidelines

1. One pull request per feature or fix — keep it focused
2. Write a clear PR title and description explaining what and why
3. Link the related issue if one exists (e.g. `Closes #42`)
4. Make sure the app still runs after your changes
5. Don't include unrelated changes or formatting fixes in the same PR

### Branch naming

| Type | Format | Example |
|------|--------|---------|
| Feature | `feat/name` | `feat/multi-account` |
| Bug fix | `fix/name` | `fix/upload-crash` |
| Docs | `docs/name` | `docs/readme-update` |
| Refactor | `refactor/name` | `refactor/storage-handler` |

## Code Style

- Use **2 spaces** for indentation (no tabs)
- Use `const` / `let`, never `var`
- Keep functions small and single-purpose
- Add comments for non-obvious logic
- File names: `camelCase.js`

No linter is enforced yet, but clean, readable code is appreciated.
